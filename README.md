# GitHub Stats Hack

Проект для генерации фиктивных коммитов на весь 2025 год для улучшения статистики на GitHub.

## Описание

Этот проект автоматически создает Git-коммиты с датами с 01.01.2025 по 31.12.2025 с реалистичным паттерном активности:

- **Выходные (суббота, воскресенье)**: больше коммитов (3-5 в день)
- **Рабочие дни (понедельник-пятница)**: меньше коммитов (1-3 в день)
- **Пропуски**: 1-2 рабочих дня в неделю без коммитов, иногда выходные тоже пропускаются

Все коммиты имеют правильные даты автора и коммитера, что позволяет улучшить визуализацию активности на GitHub.

## Использование

### Генерация коммитов

1. Убедитесь, что у вас установлен Python 3
2. Запустите скрипт генерации:

```bash
python3 generate_commits.py
```

3. После завершения генерации, вы можете добавить удаленный репозиторий и запушить коммиты:

```bash
git remote add origin <your-repo-url>
git push -u origin main
```

### Очистка истории коммитов

Если нужно начать заново, используйте скрипт для очистки истории:

```bash
python3 clear_history.py
```

⚠️ **Внимание**: Этот скрипт удалит всю историю коммитов, но сохранит все файлы в рабочей директории.

### Просмотр статистики

Для визуализации статистики в стиле GitHub:

1. Сгенерируйте данные из Git репозитория:

```bash
python3 get_stats.py
```

2. Откройте HTML файл в браузере:

```bash
open stats.html
# или просто откройте stats.html в любом браузере
```

Страница покажет:
- Календарь коммитов в стиле GitHub
- Общую статистику (всего коммитов, дней с активностью, максимальное количество коммитов в день)
- Текущую серию дней с коммитами
- Интерактивные подсказки при наведении на квадратики

### Тестирование верстки

Для автоматизированного тестирования верстки доступны два скрипта:

#### 1. Простая проверка структуры (без браузера)

Быстрая проверка HTML/CSS структуры без запуска браузера:

```bash
python3 test_layout_simple.py
```

#### 2. Полное тестирование с браузером (Playwright)

Для полноценного тестирования с запуском браузера и проверкой интерактивных элементов:

1. Установите зависимости:

```bash
pip install -r requirements-test.txt
playwright install chromium
```

2. Запустите тесты:

```bash
python3 test_layout.py
```

Скрипт автоматически:
- Откроет `stats.html` в браузере
- Проверит все элементы интерфейса
- Протестирует адаптивность (мобильная версия)
- Проверит работу tooltip при наведении
- Создаст скриншот страницы
- Выведет подробный отчет о результатах

## Структура проекта

- `generate_commits.py` - основной скрипт для генерации коммитов
- `clear_history.py` - скрипт для очистки истории коммитов
- `get_stats.py` - скрипт для получения статистики из Git
- `stats.html` - HTML визуализация статистики в стиле GitHub
- `stats.json` - файл с данными статистики (генерируется автоматически)
- `test_layout.py` - автоматизированное тестирование верстки с браузером
- `test_layout_simple.py` - простая проверка структуры HTML/CSS
- `requirements-test.txt` - зависимости для тестирования
- `src/` - директория с исходным кодом
- `docs/` - документация
- `config/` - конфигурационные файлы
- `scripts/` - вспомогательные скрипты

## Настройки

В файле `generate_commits.py` можно настроить:

- `WEEKEND_COMMITS_MIN/MAX` - количество коммитов в выходные (по умолчанию 3-5)
- `WEEKDAY_COMMITS_MIN/MAX` - количество коммитов в рабочие дни (по умолчанию 1-3)
- `WEEKEND_SKIP_CHANCE` - вероятность пропуска выходного (по умолчанию 15%)
- `WEEKDAY_SKIP_DAYS_PER_WEEK` - количество пропущенных рабочих дней в неделе (по умолчанию 1-2)

## Примечание

Этот проект создан в образовательных целях. Используйте на свой страх и риск.
